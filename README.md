# POWER-MODULE

DCDC 电源模块集合仓库

## 项目说明

本仓库用于存放各类 DCDC 电源模块的设计资料（原理图、PCB、BOM、测试文档等）。  
主分支 `main` 作为项目索引页，不直接存放具体模块的设计文件。各模块请按规范创建独立分支。

## 使用方式

1. 克隆本仓库
2. 切换到对应模块分支获取资料
3. 主分支 `main` 仅包含目录索引及上传须知

## 上传须知

- DCDC 模块按**用途、尺寸、功率、输出接口**等维度区分，不同版本请分别管理。
- **新模块**：基于 `main` 创建新分支，分支名使用格式 `module/模块名称`。
- **改进已有模块**：直接在该模块所在分支上提交更新，并**必须附带修改文档**（如 `CHANGELOG.md` 或修改说明）。
- 所有分支上传完成后，请在本 README 的“模块目录”章节同步更新条目，说明模块位置及主要变更。

## 分支管理规范

- `main`：只读索引分支，仅维护本 README，不存放设计文件。
- `module/*`：具体模块分支，每个分支独立演进。
- 禁止在 `main` 分支直接提交设计文件。
- 合并请求（Pull Request）仅用于同步目录更新，不跨分支合并非相关代码。

## 电源模块分支命名规范

### 适用范围

本规范适用于本仓库所有 DCDC 电源模块的分支命名，便于检索、分类和自动化处理。

### 基本格式

```
<模块类型>/IN<输入电压>[<输入连接器>]_OUT<输出电压>[<输出连接器>]
```

- **路径分隔符**：正斜杠 `/`，表示模块类型下的具体项目。
- **字段连接**：输入部分与输出部分用下划线 `_` 连接。
- **方括号 `[]`**：表示可选字段，可省略。
- **竖线 `|`**：表示“或”关系，用于枚举多个可能取值。

### 模块类型（必填）

| 类型标识 | 含义 |
|---------|------|
| `DCDC`  | 开关型升降压模块（可升可降） |
| `BUCK`  | 开关型降压模块 |
| `BOOST` | 开关型升压模块 |
| `LDO`   | 线性稳压器模块 |

### 输入部分

#### `<输入电压>`

- 固定电压：直接写数值加 `V`，如 `12V`、`5V`
- 电压范围：`{最小值}V-{最大值}V`，如 `5.5-24V`、`9-36V`
- 多个不连续电压：用竖线 `|` 分隔，如 `5V|12V|24V`（少见）
- **多路独立输入**：如需表示多个相同类型的独立输入，可在前面加上 `[数字]x`，例如 `2x12V` 表示两路独立的12V输入。不同路的输入类型用 `|` 分隔。

#### `[<输入连接器>]`（可选）

不写明连接器时默认为焊接/贴片方式（用户自行接线）。常用标识：

| 标识 | 含义 |
|------|------|
| `DC5521` | DC 电源插座，外径 5.5mm 内径 2.1mm |
| `DC5525` | DC 电源插座，外径 5.5mm 内径 2.5mm |
| `T` | T 型航模接头（香蕉头） |
| `PIN` | 排针/排母（2.54mm 间距） |
| `ZH` | ZH 连接器（1.5mm 间距） |
| `PH` | PH 连接器（2.0mm 间距） |
| `XH` | XH 连接器（2.54mm 间距） |
| `USBA` | USB-A 母座 |
| `USBC` | USB-C 母座 |
| `SOLDER` | 焊接式/贴片式（无连接器） |

### 输出部分

#### `<输出电压>`

输出可以是一路或多路，多路之间用竖线 `|` 分隔（不含空格）。每路格式为：

```
[<路数>x]{电压值}[{电流值}]
```

- **路数**：默认省略表示1路。多路相同规格的输出，用 `{数字}x` 前缀，如 `2x5V1A`。
- **电压值**：
  - 固定电压：`5V`、`12V` 等（注意 `3V3` 表示 3.3V）。
  - 电压范围：`3-12V`（表示连续可调）。
  - 多个可选离散值：用方括号括起，内部用竖线分隔，如 `[3V3|5V|9V|12V]`。
- **电流值**（可选）：直接写在电压后面，如 `5V3A`。若无，默认由模块设计决定。

对于“每路可从多个固定电压中选择”的模块，写成 `2x[3V3|5V|7V|9V]3A`，表示两路独立输出，每路可选 3.3V、5V、7V 或 9V，最大 3A。

对于连续可调，写成 `1x3-12V2A` 表示一路连续可调 3-12V，2A。

#### `[<输出连接器>]`（可选）

连接器标识同输入部分，可省略（默认为焊接点）。若输出有多路，连接器写在对应电压之后，格式为 `电压[连接器]`。

### 完整示例：

#### 示例1

```
BUCK/IN5.5-30V[SOLDER]_OUT5V3A[SOLDER]
```

> **说明**：BUCK 降压模块，输入电压范围 5.5-30V，采用焊接方式连接；输出 5V/3A，也是焊接方式。

---

#### 示例2
```
BUCK/IN5.5-24V[DC5521|T]_OUT5V3A[USBA]|[3V3|7V|9V|10V]3A[PIN|ZH]
```

> **说明**：BUCK 降压模块，输入电压范围 5.5-24V，输入接口支持 DC5521 或 T 型航模接头（两种可选）。  
> 输出共两路（用 `|` 分隔）：  
> - 第一路：固定 5V/3A，通过 USB-A 接口输出。  
> - 第二路：可编程输出，可选电压为 3.3V、7V、9V 或 10V，电流 3A，该路输出接口支持排针（PIN）或 ZH 1.5mm 连接器。

#### 示例3

```
DCDC/IN9-36V[XH]_OUT2x5V3A[USBC]
```
> **说明**：DCDC 升降压模块，输入 9-36V，使用 XH 连接器；输出两路完全相同的 5V/3A，均使用 USB-C 接口。

#### 示例4（多路不同输出）

```
BOOST/IN3.7V[SOLDER]_OUT5V2A[USBC]|12V1A[PIN]
```
> **说明**：BOOST 升压模块，输入通过焊接方式连接；输出两路不同的电压：一路 5V/2A USB-C，另一路 12V/1A 排针。

### 检索方法

由于分支名采用结构化格式，可使用正则表达式快速查找模块。例如：

| 需求 | 正则表达式（匹配分支名） |
|------|-------------------------|
| 所有 BUCK 模块 | `^BUCK/` |
| 带 USB 输出的模块 | `OUT.*USBA|USBC` |
| 输入带 DC5521 的模块 | `IN.*\[DC5521\]` |
| 两路输出的模块 | `OUT.*2x` 或 `OUT.*\|.*\|` |
| 输出电压包含 12V 的模块 | `OUT.*12V` |
| 使用 XH 连接器的模块 | `\[XH\]` |

在 Git 命令行中查看匹配的分支：
```bash
git branch -r | grep -E "BUCK/.*OUT.*USBA"
```
